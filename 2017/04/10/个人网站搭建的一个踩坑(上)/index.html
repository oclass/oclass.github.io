个人网站搭建的一个踩坑(上)

　　嗯，距离刚搭建完个人网站已经二十又一日了。其实我也很想早早就把这篇搭建过程早早写出来的啊，无奈忙啊摔（才不是什么懒癌发作…)

　　ps，然后写了个开头又决定将过程分成两篇blog…

　　pps，关于下面的某些网站可能要自备梯子，这里blog主不给梯子，跑得快也不给。

　　好了，关于本blog主的个人习惯暂时按下不表，以下开始正文：

　　搭建个人网站的想法最开始源于希望有一个放自己拍的照片的想法，然后了解到大概有这么几种集成的网站开发工具：WIX、Wordpress、Hexo.

WIX:

　　首先Wix属于最简单的那种，整个网站的设计都是基于html5拖拽的可视化操作方式，可以说是对小白来说最最友好的方式了。Wix自身有很多挺漂亮的模板，而且有很多是非常适配作品展示功能的，绑定自定义域名也是可以的。它还能免费使用！遗憾的是wix在国外，国内网速有一点慢，当然这不是我弃用的主要原因，这个暂时不说。

　　和Wix一类的国外的还有Squarespace、Format等，后面两个定位更高一点，也更适合作为作品网站，当然费用也更高。国内的有上线了、起飞页等，体验嘛，操作上十分类似，模板上可能会差一个档次？吧。有免费和付费两种形式。免费版都会有建站工具的水印，然后也不能绑定自己的域名。

　　上面提到我弃用Wix主要就是水印和域名的问题。基于不知道什么时候能更新一点东西也暂时没有收入来源，blog主还是不太愿意每月付十来刀去维持网站的。然后blog主就转投到了Wordpress的大家庭。

Wordpress：

　　Wordpress相较前一类型的自助建站工具复杂一点，需要有一点计算机相关的认知吧，不过对于blog主这种简单的要求也就是一点点而已，不懂也无所谓，跟着教程就完了(好吧，好像后面说的hexo也只是跟着教程= =，拍飞)。

　　Wordpress其实是一个软件，自带了后台管理、用户管理等功能，还能通过安装插件实现更多的内容。很多网站都是“自豪地采用Wordpress”，(/≧▽≦)/。知乎上面有个问题>>>有哪些知名的网站是用wordpress搭建的？,有兴趣可以了解一下。

　　用Wordpress建站其实也蛮简单的，就以下几步吧：

  1. 找一个虚拟主机

  2. 注册一个域名

  3. 将域名与虚拟主机绑定

  4. 将Wordpress上载到虚拟主机中

    ​

　　更具体一点：

  1. 虚拟主机，blog主用的是000webhost，一个国外的免费虚拟主机，有多少容量我忘记了…记得挺多的，肯定是能支撑个人网站的了。类似的还有Openshift，不过它当时在更新，然后只能用github账号申请试用，还要等审核。blog主没有空没等审核通过就注册000webhost了。听说阿里云还是腾讯云有个1元租一年虚拟主机的项目，在校生可以申请一下。blog主因为厌恶备案，同时那时候已经用上hexo+github了，就没有去了解了。(关于备案：一般来说要绑定域名，国内的虚拟主机提供商都要求必须备案，而国外的则不用，当然国外的访问速度会比国内慢，以及，英文…blog主那两天看了大学以来最多的英文，(￣︶￣)↗　)

    ​

  2. 域名免费的可以到freenom申请，后缀有.ml/.tk/.ga/.cf/.gq几种，一年期过了继续+1s就可以了。(不过听说有被freenom自行注销的风险，blog主也不清楚，毕竟blog主现在是RMB玩家了，笑)。付费的可以到万网、godaddy、西部数据等网站，blog主是在西部数据注册的，.site域名，挺便宜的(其实我想要.me的摔（╯‵□′）╯︵┴─┴… )

    ​

  3. 然后是将域名与虚拟主机绑定，首先你需要去解析域名，blog主用的DNSPOD，速度快不解释。然后到虚拟主机上进行相关的设置就好了。这部分不懂直接google，我不记得太清楚了。

    ​

  4. 然后就是将Wordpress装到虚拟主机上，下载、解压、上传虚拟主机指定目录，完成！以000webhost为例，噢，对了，000webhost自带有一键安装Wordpress的功能，但blog主并没有成功，大概是因为是免费账户的受害者。嗯，从Wordpress下载安装包以后直接用File manager将它上传到000webhost的根目录，再用网站自带解压进行解压（这样比较快，就像香港记者一样快！），然后创建Database，修改wp-config.php文件再上传就好，具体>>>免费空间如何安装WordPress。

    ​

  5. 还有关于000webhost不懂得地方click>>>help，关于其他不懂的google，zhihu。
