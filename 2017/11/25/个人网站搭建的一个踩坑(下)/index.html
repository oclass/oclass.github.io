关于hexo搭建GitHub静态页面

首先，配置GitHub

  1. 注册GitHub以后 create a new repository ,在repository name填入你的username，然后你的blog域名就是yourname.github.io（yourname就是你的username）

然后，配置环境

  1. 安装 Node.js ,参考相关

  2. 安装Git 官网 镜像站
    安装完成以后终端执行git --version检查是否安装成功

  3. 安装Hexo , 当Node.js和Git安装完成以后，可以用cmd安装Hexo , 推荐使用cmder
    $ npm install -g hexo-cli
    等待，直至安装完成
    以及一些插件

　环境安装完成。

Third，建立本地blog文件夹并部署到GitHub

  1. 随意在某个硬盘建立你的blog文件夹

  2. 右键Cmder，cmder here,输入
    hexo init blog
    成功建立后会显示 INFO Start blogging with Hexo!

  3. 继续执行
    hexo g
    hexo s
    然后用浏览器打开本地预览，嗯，这就是最初始状态的blog页面，不过是在本地的

  4. 重新打开cmder , 执行
    ssh-keygen -t rsa -C your_email@youremail.com
    your email@youremail.com就是你GitHub的注册邮箱，一路enter下去，提示成功后会在某处生成id_rsa.pub文件，复制里面的所有信息，（可以用sublime text打开，记事本应该也可以）

  5. 登录github , 右上角 头像 -> setting ->SSH and GPG keys -> New SSH key . 粘贴，填好title并 Add SSH key

  6. 验证是否成功。
    ssh -T git@github.com
    第一次会提示continue or not , yes之后会见到成功的提示。

  7. 在本地的blog目录下，找到_config.yml，修改其中的参数，参数的含义，emmm , 看英文含义。
    ps , 每个参数的：后都要加一个空格

  8. 继续修改_config.yml

　然后保存_config.yml , cmder执行
　hexo clean
　hexo g
　hexo d
　跳出GitHub登录，直接登录就好了。
　然后就部署完成了。然后一些常见问题参考 >>> hexo的坑one,hexo的坑two

Forth , 发表文章

　创建的文章在source_posts\下，用markdown语法编写，参见markdown语法
　Okay，到此你已经可以在你的个人网站上写东西了

　Emmmm , 也可以继续完善

Fifth，网站主题

  1. 找主题，>>> 知乎

  2. Git clone 主题的地址

  3. 修改_config.yml中的theme属性,然后更新
    Cd themes/themesname
    Git pull

　Eg:
　git clone https://github.com/joyceim/hexo-theme-apollo.git themes/apollo
　更改_config.yml中theme的参数为apollo
　cd themes/apollp
　git pull
　完成！

Sixth，在menu上添加新的分类页面

  1. Hexo new page “title”

  2. Source下会生成一个title的目录，里面有个index.md，修改这个页面就可以了，写法和文章一样
    参见>>> 相关

Seventh，绑定个人域名

  1. 关于买域名，及解释域名参照上一篇踩坑记录

  2. 在source下创建CNAME，写入自己的域名

  3. hexo g
    hexo d
    完成！

  4. 只显示个人域名,
    打开你的GitHub仓库，找到setting（在仓库下面那栏的最右边），click,
    找到GitHub Pages , 在Custom domain栏填写你的域名，save ,
    finish!

The Final !

  1. 推荐使用Typora作为markdown的编辑器，无他，好看尔。

  2. 或者MarkDownEditor
